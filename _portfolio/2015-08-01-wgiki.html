{ HOME }

The home page lists each division of WGI and its associated color in a stacked card-style format. Clicking a card will take you to the page for that division. At the bottom of the home page (and every page) is a footer containg the names of the content managers for each division.

{% include browser.html url="wgiki.com" image="portfolio_wgiki" extension="jpg" alt="| Home" scroll="scroll" %}

{ NAVIGATION }

The WGIki has a few navigational components. The menu icon in the top right opens the side nav with some user info as well as division links. Categories for a division appear if you're under that division's umbrella of pages. Clicking the star in the top right opens the favorites drawer with quick links. Finally, fixed in the bottom right are floating buttons that open for feedback or help forms.

{% include browser.html url="wgiki" image="portfolio_wgiki_navigation" extension="jpg" alt="| Navigation" %}

{ DIVISIONS }

At the top of the resource hierachy is the division. Each division within WGI has its own categories and resources that it maintains. On the main page for each division, links to all its categories are listed with a brief description. Oh, and just for a little added flare, each division also has a unique color that is used thematically throughout its pages (notice the nav bar and cards).

{% include browser.html url="wgiki.com/division" image="portfolio_wgiki_divisions" extension="jpg" alt="| Divisions" %}

{ CATEGORIES }

Next up in the hierachy are categories. Individual resources are assigned categories, and the main page for each category lists all the resources that make it up. A short description is provided to help give users an idea of what they'll find here. Resources are presented in a tab-like list with icons and other useful information to let users know what they're in for. Hovering over a resource makes it scale up (slightly).

{% include browser.html url="wgiki.com/category" image="portfolio_wgiki_categories" extension="jpg" alt="| Categories" %}

{ RESOURCES }

Finally, at the bottom of the hierachy are individual resources. These can come in a variety of formats (single files, multiple files, external links, self-composed articles, etc.), and an edit button is available to content managers for quick access to the backend of a page.

{% include browser.html url="wgiki.com/resource" image="portfolio_wgiki_resources" extension="jpg" alt="| Resources" scroll="scroll" %}

{ SEARCH }

The search functionality of the WGIki scours all resources and returns a list of the most relevant ones (using Relevanssi. The results are listed just like resources on a category page, with a little additional information about where to find them specifically.

{% include browser.html url="wgiki.com/search" image="portfolio_wgiki_search" extension="jpg" alt="| Search" %}
